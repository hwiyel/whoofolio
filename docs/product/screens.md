# 화면 구조와 정보 설계

## 상위 네비게이션

- `개요`
- `거래`
- `보고서`
- `카테고리`
- `계정`
- `순자산`
- `설정`

## 1. 개요

첫 화면은 10초 안에 이번 달 상태를 파악하게 해줘야 한다.

- 이번 달 수입, 지출, 순현금흐름
- 전월 대비 변화
- 지출 상위 카테고리
- 큰 거래 하이라이트
- 계정별 잔액 스냅샷
- 최근 이상 징후 또는 급격한 변화

## 2. 거래

가장 중요한 탐색 화면이다.

- 테이블 중심 레이아웃
- 강한 필터: 기간, 계정, 자산, 카테고리, 키워드, 금액 범위
- 저장된 뷰: `이번 달`, `고정비`, `구독`, `큰 지출`
- 행 클릭 시 원본 맥락과 관련 데이터 확인

## 3. 카테고리

이 화면은 “얼마 썼는가”보다 “어떻게 쓰고 있는가”를 설명해야 한다.

- 월별 카테고리 순위
- 최근 N개월 카테고리 추이
- 고정비 / 변동비 구분
- 하위 카테고리 또는 원거래로 drill-down

## 3-1. 보고서

리포트 화면은 여러 차트를 늘어놓는 곳이 아니라, 해석이 필요한 질문을 묶는 곳이다.

- `월간 리포트`
- `카테고리 리포트`
- `계정 흐름 리포트`
- 모든 리포트는 거래 목록으로 이어져야 한다

## 4. 계정

생활 자금 흐름과 자산 보유 현황을 분리해서 보게 한다.

- 계정별 현재 잔액
- 월별 잔액 변화
- 계정별 입금 / 출금
- 이체 비중이 큰 계정 구분

## 5. 순자산

v1에서는 욕심내지 않고 가볍게 시작한다.

- 자산 / 부채 요약
- 순자산 추이
- 현금 / 투자 비중

## 조회 화면 원칙

- 기본 기간은 `이번 달`
- 모든 요약 블록은 필터된 거래 목록으로 연결한다
- 읽기 흐름을 깨는 깊은 모달 체인은 피한다
- v1에서는 읽기 중심 화면에 편집 UI를 섞지 않는다

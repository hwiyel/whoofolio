# 보고서 설계

## 목표

`보고서`는 예쁜 차트를 늘리는 기능이 아니라, 후잉 데이터를 더 빨리 해석하게 해주는 읽기 화면이어야 한다.  
1차 버전에서는 `이번 달이 어떤가`, `어디에 얼마나 썼나`, `어떤 흐름이 변했나`에 답하는 데 집중한다.

## 1차 리포트 범위

### 1. 월간 리포트

가장 먼저 구현할 보고서다.

- 대상 질문
  - 이번 달 수입, 지출, 순현금흐름은 어떤가
  - 전월 대비 얼마나 늘거나 줄었나
  - 이번 달 큰 거래와 상위 지출은 무엇인가
- 핵심 블록
  - 수입 / 지출 / 순현금흐름 요약
  - 전월 대비 변화
  - 상위 지출 카테고리
  - 큰 거래 목록

### 2. 카테고리 리포트

지출 구조를 읽는 화면이다.

- 대상 질문
  - 어떤 카테고리에 돈이 가장 많이 나갔나
  - 최근 N개월 동안 어떤 카테고리가 늘었나
- 핵심 블록
  - 카테고리별 지출 순위
  - 월별 카테고리 추이
  - 상위 카테고리 drill-down

### 3. 계정 흐름 리포트

계정별 잔액보다 흐름 설명에 초점을 둔다.

- 대상 질문
  - 어떤 계정이 자금 이동의 중심인가
  - 기간 중 어떤 계정으로 돈이 몰렸나
- 핵심 블록
  - 계정별 입금 / 출금 합계
  - 계정 간 이동 상위 흐름
  - 특정 계정으로 필터된 거래 진입

## 화면 원칙

- 기본 기간은 `1M`
- 모든 수치는 거래 목록으로 drill-down 가능해야 한다
- 차트보다 비교 가능한 표와 순위를 우선한다
- v1에서는 리포트마다 질문이 분명해야 한다

## 구현 순서

1. `월간 리포트`
2. `카테고리 리포트`
3. `계정 흐름 리포트`

## API / 데이터 요구사항

- 월간 집계 API
  - 수입, 지출, 순현금흐름
  - 전월 비교
- 카테고리 집계 API
  - 기간별 카테고리 합계
  - 월별 추이
- 계정 흐름 API
  - 계정별 입출금
  - 계정 간 이동 합계

## 하지 않을 것

- v1에서 예산 시뮬레이터를 넣지 않는다
- AI 요약을 먼저 넣지 않는다
- 보고서마다 다른 기간 체계를 두지 않는다

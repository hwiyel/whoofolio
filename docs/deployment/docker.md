# Docker 배포

## 배포 목표

`whoofolio`를 개인 서버에 올려 두고, 브라우저로 접속하는 사설 웹 앱 형태로 운영한다. 업그레이드와 백업은 단순해야 하고, 후잉 원본 데이터를 망가뜨릴 가능성은 낮아야 한다.

## 권장 구성

- `reverse-proxy`: 기존 Nginx Proxy Manager 사용
- `app`: whoofolio 웹 + API
- `db`: Postgres

초기 버전에서는 프론트와 API를 하나의 컨테이너로 묶고, Nginx Proxy Manager가 `app:8080` 또는 호스트의 `8080` 포트로 연결되게 두면 충분하다.

## 환경 변수

- `WHOOING_API_KEY` 또는 OAuth 관련 자격 정보
- `DATABASE_URL`
- `APP_BASE_URL`
- `SESSION_SECRET`
- `SYNC_SCHEDULE`

비밀값은 Git에 넣지 않는다. Docker secrets, Git 외부 `.env`, 또는 서버의 비밀 관리 방식을 쓴다.

## 동기화 전략

- 최근 데이터는 몇 분 단위로 주기 동기화
- UI에서 수동 `지금 새로고침` 지원
- 오래된 기간의 전체 재동기화는 명시적으로 요청할 때만 수행

## 백업 정책

원장 자체는 후잉이 기준이지만, 아래 데이터는 백업 대상이다.

- 저장된 필터
- 화면 설정
- 집계 캐시
- 인증 설정

즉, DB와 앱 설정은 백업한다. 후잉 원장 전체를 별도 원본처럼 중복 관리하려고 하지는 않는다.

## 보안 원칙

- 반드시 HTTPS 뒤에 둔다
- 서버가 외부에 열려 있다면 간단한 인증을 추가한다
- 실제 필요가 생기기 전까지 단일 사용자만 지원한다
- 미래의 쓰기 기능은 기본 비활성화로 시작한다

## 첫 구현 목표

1. `docker compose` 하나로 기동
2. 개인 사용자 1명
3. read-only 동기화
4. 월간 대시보드와 거래 탐색기 제공

이 범위는 작지만, 실제 만족도를 검증하기에는 충분하다.
